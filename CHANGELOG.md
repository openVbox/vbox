# Changelog

All notable changes to vbox.

Format follows [Keep a Changelog](https://keepachangelog.com/). The project is
pre-1.0; released changes are grouped by version, and upcoming work lives under
`[Unreleased]`.

## [0.1.1] - 2026-05-20

### Changed

- GitHub Release publishing now composes release notes from the matching
  changelog version section plus `SHA256SUMS`, instead of publishing
  checksums as the entire release body.

### Fixed

- Installed macOS per-app launchers no longer all reuse the base `5710`
  data port. Named app instances now get stable per-instance ports
  recorded under `.vbox/instance-ports.tsv`, and generated `.app`
  bundles write the allocated port into `Port.txt`, preventing one
  running launcher from blocking another with a controld port conflict.

## [Unreleased]

### Added

#### Transport & data plane
- Data-plane QUIC transport (UDP) on top of `quinn 0.11`, with
  `DataPlaneMode::{TcpOnly, QuicOnly, Auto}` negotiation, reliable
  streams + unreliable datagrams + multi-app channels.
- `BootstrapBundle` carrying TCP + QUIC addrs, session token, and QUIC
  cert sha256 — viewer dials the data plane directly after one
  controld RPC.
- Per-instance self-signed QUIC certificate auto-generated by
  `vbox-controld` and reused on subsequent runs.
- Wire types `ViewRequest`, `WindowEvent`, `FrameTile`,
  `PixelEncoding`, and `InputEvent::{PointerMotion, PointerButton,
  PointerScroll, Key, Text, Focus, Preedit, Resize, Close,
  ToggleMaximize}`.

#### Compositor & rendering
- Smithay-based nested Wayland compositor on the guest, exposed as
  `WAYLAND_DISPLAY=vbox-0`. Implemented protocols: `wl_compositor`,
  `wl_shm`, `xdg_wm_base`, `wl_output`, `wl_seat`.
- `xdg_toplevel` `wl_shm` buffers are converted to `RawRgba` FrameTiles
  and streamed to the host.
- Multi-toplevel routing: server tracks `HashMap<id, RemoteWindow>` and
  routes root-surface + subsurface commits per remote window id; the
  macOS viewer spawns one independent `winit` + `softbuffer` window per
  id.
- Bidirectional resize: `InputEvent::Resize` ↔ `xdg_toplevel.configure`.
- `InputEvent::{Close, ToggleMaximize}` → `xdg_toplevel.close /
  set_maximized` (host close / maximize buttons + edge snap).
- `InputEvent::PointerScroll` → `wl_pointer.axis` (mouse wheel).
- `WindowEvent::{Minimized, MoveRequested}` for guest → host requests.
- `WindowEvent::Created` carries `app_id` for the host launcher mapper.

#### Input & IME
- Pointer / click / focus / basic-key forwarding from the macOS viewer
  into the guest via the Wayland seat.
- Minimal `zwp_text_input_v3` manager in the server compositor;
  `InputEvent::Text` becomes `commit_string + done` and
  `InputEvent::Preedit` becomes `preedit_string + done` when the
  focused app enables it.
- ASCII key-synthesis fallback for apps without text-input-v3.
- Named-key path: Backspace, Enter, Tab, arrows, Delete.
- Client-side 2벌식 composer assembles compatibility jamo
  (`ㅇㅏㄴㄴㅕㅇ`) into syllables (`안녕`) before sending preedit /
  commit.

#### Control plane
- `vbox-controld` daemon: single source of truth for instance / app
  pid lifecycle. Wire kinds `300 / 301` carry
  `RpcRequest` / `RpcResponse`.
- `RpcMethod::{Authenticate, Status, StartInstance, StopInstance,
  LaunchApp, WaitSocket}`.
- Plain-token authentication: shared secret in
  `~/.vbox/controld.token`, sent over `ssh -L 5711`.
- mTLS authentication: `vbox tls-bootstrap` issues CA / server /
  client certs; `VBOX_TLS_*` env activates the client; mTLS skips the
  token check and removes the ssh -L hop.
- `vbox controld-install` / `controld-uninstall` / `controld-status`
  manage controld as a systemd user unit.
- `vbox controld-install --controld-bin PATH --server-bin PATH` flags
  to override the systemd `ExecStart` binary paths (for prebuilt
  tarball or system-package installs). When the flags are omitted the
  install script tries `command -v vbox-controld / vbox-server` on the
  guest first, then falls back to `$guest_dir/target/release/...`.

#### Tooling & distribution
- Workspace crates: `proto`, `server`, `controld`, `client`, `vbox-cli`.
- `cargo build --release` succeeds on macOS host and
  `aarch64-unknown-linux-gnu` guest.
- Debug CLI: `vbox click`, `vbox type`, `vbox key`, `vbox logs input`.
- Homebrew Cask (`brew install --cask vbox`) ships a universal ad-hoc
  signed CLI tarball; per-app `.app` launchers are generated locally
  by `vbox install-apps`.
- `scripts/install-linux.sh` — single-URL guest installer:
  `curl -fsSL https://raw.githubusercontent.com/openVbox/vbox/main/scripts/install-linux.sh | sh`.
  Auto-detects arch, installs `libxkbcommon` via apt / dnf / pacman /
  zypper, fetches the matching prebuilt tarball into `/usr/local/bin/`,
  writes the systemd user unit, enables it, turns on
  `loginctl enable-linger`, and prints the exact macOS-side command.
  `VBOX_VERSION` / `VBOX_PREFIX` / `VBOX_DIR` / `VBOX_PORT` env vars
  override the defaults.
- Multilingual README (English / 한국어 / 日本語 / 简体中文 / Español)
  with a language switcher.
- Localhost ping RTT ≈ 0.1 ms.

### Fixed
- `vbox controld-install` used to hard-code `$guest_dir/target/release/`
  for the systemd `ExecStart`, which meant prebuilt-tarball
  deployments (binaries under `/usr/local/bin/`) silently failed to
  start. The install script now resolves the binary path via the new
  `--controld-bin / --server-bin` flags, then `command -v` on the
  guest, and only falls back to the cargo path for from-source
  installs.
- Composite `wl_shm` frames over a white background so transparent
  regions don't render as solid black.
- Seat / text-input enter state was lost after a mouse click + focus
  change — now preserved.
- Cmd / Ctrl combinations no longer leak into guest text input.
- README architecture diagram corrected: `vbox-controld` lives inside
  the Linux guest (not between host and guest), and the data plane has
  no TLS path (only QUIC or ssh-tunnelled TCP).
